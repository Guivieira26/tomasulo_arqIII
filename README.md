# tomasulo_arqIII

O trabalho 2 é o desenvolvimento de um simulador para fins didáticos (qualquer linguagem) do algoritmo de Tomasulo.

Considerem instruções MIPS fins de ilustração em simulação.

O simulador deve apresentar métricas de desempenho, tais como: IPC, quantidade de ciclos gastos, quantidade de ciclos de bolha, etc.

O simulador deve possuir uma interface gráfica capaz

de auxiliar o estudante no entendimento do algoritmo e consequente

arquitetura superescalar. Opção de execução passo a passo deve propiciar

ao usuário a capacidade de entender o que acontece com as instruções em

diferentes momentos, tais como despacho, execução e commit.

Lembrar que esta versão de simulador deve possuir buffer de reordenamento e suportar especulação de instruções de desvio condicional.
